# 增量式大纲拓展 Agent 需求分析

## 核心诉求

设计一个增量式拓展小说大纲的 expert agent，替代现有的 `character.py` 和 `outline.py`。

核心模式：
1. 初始轮：构建核心人物、世界观、故事线、总纲
2. 迭代轮（N轮，用户指定）：每轮加入人物、故事线，完善世界观，填充总纲
3. 每轮结束后评估
4. 循环结束后产出：完整人物设计、故事线、世界观、总纲和卷纲

## 现有架构（被替代部分）

- `character.py`：独立 sub-agent，生成 `character/*.md`，内置 evolution 评估（最多3轮）
- `outline.py`：独立 sub-agent，生成 `outline.md`，内置 evolution 评估（最多3轮）
- 两者线性串联：meta agent 先调 character，再调 outline
- 世界观和故事线没有独立实体，故事线嵌在 outline.md 中（主线/次线/交织点）

## 新增概念（现有系统中不存在）

- **世界观**：独立的创作维度，当前系统中无对应产物
- **故事线**：从 outline.md 中拆出成为独立实体（当前主线/次线/交织点嵌入 outline.md）
- **卷纲**：当前系统中 detail_outline 接近但不同，需进一步澄清

## 已确认决策

### D1: 替换边界
- 替代 `character.py` + `outline.py`
- **卷纲 ≠ detail_outline**。卷纲是"卷级大纲"——比总纲细一层，按"卷"（多章一组）拆分，每卷包含人物、主线走向、关键转折，但不到章节级别
- 产出管线：总纲 → 卷纲 → （后续 detail_outline 拆到章节）
- detail_outline.py、writing.py 不在本次替换范围

### D2: 循环控制
- **Meta agent 外循环**：meta agent 每轮调用本 agent 一次，传入"第 X 轮、已有产物"，agent 做一轮扩展+评估后返回，meta agent 决定是否继续

### D3: 总纲与故事线的分工
- **总纲 = 结构蓝图**：只管宏观叙事结构（几幕、节奏曲线、卷划分）、各故事线的交汇编排、整体主题/基调
- **故事线 = 独立实体**：角色动机、弧线走向、因果链等具体内容归故事线实体

### D4: 评估模型
- **统一评估**：每轮结束后一个评估 agent 整体读取所有维度产物，从跨维度一致性角度评估。一个评估，一份报告。

### D5: 卷纲生成时机（已变更）
- **从第一轮开始就包含卷纲**：每轮迭代都同时产出人物、故事线、世界观、总纲和卷纲。卷纲跟其他维度同步增长。
- 第一轮产出初步的卷划分（可能只有1卷的基本结构），后续轮次随故事丰富扩展卷数和卷纲内容。

### D6: 每轮扩展策略
- **用户指定 + Agent 补充**：meta agent 转发用户的扩展方向（优先），agent 根据评估建议补充其他维度（兜底）。用户没给方向时，agent 完全自主决定。

### D7: 评估低分处理
- 低分轮不计入"跳过"，而是**下一轮优先修复评估指出的问题**，修复完成后继续扩展。
- 评估在本轮扩展后执行，如果评估分数低于阈值，agent 在本轮内尝试修订（沿用 evolution 模式，最多修订 3 轮）；修订后仍低分则标记质量风险返回，由 meta agent 在下一轮处理。

### D8: 第一轮最小可用产出
- **最低门槛**：至少 1 个主角 + 1 条主线 + 基础世界观框架（时代/空间/冲突根源）+ 总纲骨架（故事核心+结构骨架）+ 初步卷划分
- 其他维度可以留占位符（"待后续轮次补充"）

## 已确认要素设计

### 人物要素
| 要素 | 说明 |
|---|---|
| 基本信息 | 姓名、年龄、性别、身份/职业 |
| 角色定位 | 主角/配角/反派/功能性角色；叙事结构功能（推动者/回应者/催化剂/对立面） |
| 外貌特征 | 长相体态、穿着打扮、标志性细节（具体、有画面感） |
| 性格特质 | 核心性格标签 + 行为标识（口头禅、习惯动作、决策模式）——用行为定义性格，不写心理描写 |
| 核心欲望与恐惧 | Want（外在目标）vs Need（内在需求）；核心恐惧 |
| 角色弧光 | 起点状态 → 转变契机 → 终点状态（弧度与故事体量匹配） |
| 能力与局限 | 擅长什么、不擅长什么、致命弱点 |
| 关系网络 | 与各角色的关系类型（亲疏/对立/依赖）+ 动态变化方向 |

### 世界观要素
| 要素 | 说明 |
|---|---|
| 时代与背景 | 什么时代、什么社会形态、科技/文明水平 |
| 地理与空间 | 故事发生的空间范围、关键地点及其功能 |
| 社会结构 | 阶层/阶级、权力分配、重要组织/势力 |
| 核心规则 | 世界运转的底层规则（奇幻→力量体系/魔法规则；现实→社会法则/经济逻辑） |
| 文化与信仰 | 主流价值观、禁忌、传统习俗 |
| 历史脉络 | 影响故事走向的重大历史事件（只写与故事有关的） |
| 冲突根源 | 这个世界的核心矛盾——故事冲突的土壤 |

（设计原则：每个要素"与故事有关才写"，不写世界百科全书）

### 故事线要素
| 要素 | 说明 |
|---|---|
| 名称 | 这条线的标识 |
| 类型 | 主线 / 次线 / 暗线 |
| 驱动角色与动机 | 谁在推动这条线，核心动机是什么 |
| 核心冲突 | 这条线围绕的矛盾/张力 |
| 弧线走向 | 起点状态 → 终点状态（状态转变，不是情节展开） |
| 关键转折点 | 结构性 beat——定义"结构位置和功能"，不写具体场景 |
| 与其他故事线的关系 | 依赖、对立、并行、交汇——以及交汇产生的新张力 |
| 功能定位 | 这条线为什么存在（对照主题/放大张力/提供对照视角/埋设伏笔） |

存储方式：每条线一个文件，`storyline/{线名}.md`（与人物保持一致的"一个实体一个文件"模式）

### 总纲要素
| 要素 | 说明 |
|---|---|
| 故事核心 | Logline、核心主题、类型与基调 |
| 结构骨架 | 结构类型 + 选择理由 |
| 节奏曲线 | 全篇张力分布 |
| 卷划分 | 分几卷、每卷的核心结构功能 |
| 故事线编排 | 各故事线在哪一卷交汇、交织方式、交织产生的冲突 |
| 主题递进 | 主题在各阶段的深化方式 |

### 卷纲要素（循环结束后生成）
| 要素 | 说明 |
|---|---|
| 卷标题 | 这卷叫什么 |
| 卷定位 | 在整体弧线中的结构功能（承接/建立/输出） |
| 该卷涉及的故事线 | 哪些线在本卷活跃、各自走向和进度 |
| 出场人物 | 本卷核心人物 + 进入/退出状态 |
| 关键转折与事件 | 本卷必须发生的结构 beat |
| 上下卷衔接 | 与前一卷的承接点、向下一卷的推动力 |
| 该卷内部节奏 | 卷内张力曲线 |

### 统一评估维度（6维度，满分100）

| # | 维度 | 分值 | 评估什么 |
|---|---|---|---|
| 1 | 人物-故事线匹配 | 20 | 角色弧光与故事线弧线是否一致？核心欲望是否驱动核心冲突？有无游离角色/空头故事线？ |
| 2 | 世界观-故事线支撑 | 15 | 世界观能否支撑故事线涉及的力量体系/社会冲突？有无故事线假设了不存在的世界元素？ |
| 3 | 总纲编排合理性 | 20 | 交织编排是否准确？节奏曲线是否匹配？卷划分是否合理？ |
| 4 | 增量一致性 | 20 | 新增与已有之间有无矛盾？关系网络、交汇点是否自洽？ |
| 5 | 覆盖度 | 15 | 有无引用未定义的世界观元素？未创建的角色？不存在的交织点？ |
| 6 | 主题贯穿度 | 10 | 所有故事线是否指向同一核心主题？有无游离线？ |

评分：90+ 继续 / 75-89 调整后继续 / 60-74 修订后再继续 / <60 暂停修复

### 文件/目录结构

```
/
├── character/           # 人物档案，一个人物一个文件
├── storyline/           # 故事线，一条线一个文件
├── worldview.md         # 世界观（单文件，各要素高度耦合）
├── outline.md           # 总纲（结构蓝图）
├── volume/              # 卷纲，循环结束后生成
│   ├── volume-01.md
│   └── ...
└── evaluation.md        # 统一评估报告（每轮覆盖写入）
```
