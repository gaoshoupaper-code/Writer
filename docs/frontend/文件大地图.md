# frontend 文件大地图

> ⚠️ **本文档已废弃**，由 [`docs/系统心智模型.md`](../系统心智模型.md) 取代，不再更新。
> 系统的全貌、请求流转、机制深潜请看新的心智模型图。本文件保留作历史归档。

> **怎么看这张图**：frontend 是用户直接操作的网页界面（Next.js 应用），自己不做 AI 计算，
> 只负责转发请求给 executor、把结果展示出来。下方按"页面/业务组件/基础控件/工具/配置"分卡片。
> 每张卡片先告诉你"这组管什么"，再逐个列出文件和作用。

## 📄 页面层（app/）

> **这组管什么**：每个文件夹对应一个网址，里面的 `page.tsx` 就是那个网址的页面。
> Next.js 约定：`app/login/page.tsx` → 网址 `/login`。`layout.tsx` 是所有页面共享的框架。

| 文件 | 作用 |
|------|------|
| `app/layout.tsx` | 全局布局。所有页面共享的框架（顶栏、侧边栏、全局样式注入）。每个页面都套在这个布局里，保证整站视觉一致。 |
| `app/page.tsx` | **首页（/）= 主工作区**。用户登录后在这里创作——填需求、看进度、管理产出。是 frontend 最核心的页面。 |
| `app/globals.css` | 全局样式表。定义整站的 CSS 变量和基础样式。 |
| `app/login/page.tsx` | 登录页（/login）。用户输入账号密码登录，成功后跳转首页。 |
| `app/register/page.tsx` | 注册页（/register）。新用户注册（需邀请码）。 |
| `app/settings/page.tsx` | 设置页（/settings）。用户配置自己的 API key、模型等，对应 executor 的多用户隔离（每个用户用自己的 key）。 |
| `app/admin/page.tsx` | 管理后台（/admin）。管理员专用，管理用户、查看全局状态。 |
| `app/skills/page.tsx` | 技能管理页（/skills）。管理 AI 技能，对应 executor 的 platform/skills。 |

## 🧩 业务组件层（components/workspace/）

> **这组管什么**：专为"写作工作区"做的面板和控件。
> **关键认知**：这组的组件和 executor 的写作流水线**一一对应**——流水线有四个岗位，前端就有对应的四个面板展示它们的产出。

### 工作区框架组件

> **这组管什么**：工作区的整体骨架和导航——把顶栏、侧栏、主内容区拼起来。

| 文件 | 作用 |
|------|------|
| `workspace/AppShell.tsx` | 应用外壳。工作区的整体骨架，把顶栏、侧边栏、主内容区拼起来。所有工作区内容都装在这个壳里。 |
| `workspace/TopBar.tsx` | 顶栏。工作区顶部的操作栏（会话切换、操作按钮等）。 |
| `workspace/Sidebar.tsx` | 侧边栏。工作区左侧的导航/会话列表，切换不同创作会话。 |
| `workspace/SessionMenu.tsx` | 会话菜单。管理对话会话（新建、切换、删除），对应 executor 的 threads。 |
| `workspace/ConfirmDialog.tsx` | 确认弹窗。通用确认对话框，删除、覆盖等危险操作前弹出确认。 |

### 流水线进度展示组件

> **这组管什么**：展示写作流水线的实时进度，让用户看到"现在做到哪一步了"。

| 文件 | 作用 |
|------|------|
| `workspace/StageFlowView.tsx` | 阶段流视图。展示整条写作流水线的进度（故事构建→分场→正文），用 lib/stage.ts 的数据。是用户看"现在到哪一步了"的主视图。 |
| `workspace/StageTrail.tsx` | 阶段轨迹。展示已完成阶段的轨迹记录，配合 StageFlowView 展示历史。 |
| `workspace/ChainNodeIcon.tsx` | 链路节点图标。渲染 trace 链路里每个节点的小图标（agent/工具/错误各不同图标）。 |

### 写作流水线各环节面板

> **这组管什么**：每个面板对应流水线一个环节的产出展示。
| 文件 | 对应流水线 | 作用 |
|------|-----------|------|
| `workspace/ChatPanel.tsx` | 第一环·访谈 | 对话面板。访谈员和用户对话的界面，用户在这里回答访谈员的问题。 |
| `workspace/InterviewOptions.tsx` | 第一环·访谈 | 访谈选项。访谈环节给用户展示的可选项（快速选择，而不必每次打字），配合 ChatPanel。 |
| `workspace/StorylinePanel.tsx` | 第二环·故事 | 剧情线面板。展示故事建筑师产出的剧情线。 |
| `workspace/CharactersPanel.tsx` | 第二环·故事 | 人物面板。展示人物设计。 |
| `workspace/WorldviewPanel.tsx` | 第二环·故事 | 世界观面板。展示世界观设定。 |
| `workspace/DetailOutlinePanel.tsx` | 第三环·分场 | 分场大纲面板。展示分场细化员产出的大纲，用 lib/outline.ts 处理数据。 |
| `workspace/ScriptPanel.tsx` | 第四环·正文 | 剧本面板。展示正文写手产出的剧本正文，**用户最终看到的成品主要在这里**。 |
| `workspace/NovelPanel.tsx` | 第四环·正文 | 小说视图面板。用小说体裁展示正文（ScriptPanel 是剧本体裁，这是另一种呈现）。 |

### 追踪与统计组件

> **这组管什么**：展示执行轨迹（trace）和 token 消耗，对应 executor 的 trace 系统。

| 文件 | 作用 |
|------|------|
| `workspace/TracePanel.tsx` | trace 面板。展示执行轨迹详情（每一步调了谁、耗时、结果），对应 executor 的 trace 系统。 |
| `workspace/TraceChainDrawer.tsx` | trace 链路抽屉。以抽屉形式展开完整的 trace 调用链（父子 agent 关系）。 |
| `workspace/TraceChainTimeline.tsx` | trace 时间线。按时间顺序展示 trace 事件，配合 TraceChainDrawer。 |
| `workspace/TokenChartPanel.tsx` | token 用量图表。展示每次运行的 token 消耗统计（花了多少、各环节占比）。 |

### 风格与其他组件

| 文件 | 作用 |
|------|------|
| `workspace/StyleModal.tsx` | 风格管理弹窗。让用户管理创作风格（对应 executor 的 styling），选不同风格影响生成结果的文风。 |
| `workspace/ImageReviewCard.tsx` | 图像评审卡片。展示和评审图像生成结果（对应 image 域）。 |

## 🔌 工具层（lib/）

> **这组管什么**：非 UI 的逻辑——调 API、定义数据格式、工具函数。
> **最关键的是 `api.ts`（前后端通信咽喉）和 `stage.ts`（trace 转进度）**，它们是 frontend 和 executor 对接的核心。

| 文件 | 作用 |
|------|------|
| `lib/api.ts` | **和 executor 对接的统一入口（最关键）**。封装所有 HTTP 调用，统一处理认证（带 cookie）、错误、401 自动跳登录。所有要调后端的地方都走它，保证认证态一致、不漏处理错误。是前后端通信的咽喉。 |
| `lib/types.ts` | **前后端共享的数据格式定义（最关键）**。定义了剧本、工作区、线程、trace、风格等所有数据结构。和 executor 的 schemas/ 一一对应，保证两边数据格式一致。 |
| `lib/stage.ts` | 阶段流转换器。把 executor 返回的 trace（执行轨迹）转换成前端的"阶段流"数据（storybuilding/detail-outline/writing 等阶段 + 进度）。让用户看到"正在做哪一步"的可视化进度，是 trace→用户可视化的翻译层。 |
| `lib/trace.ts` | trace 数据处理工具。处理 trace 详情的解析、格式化，配合 TracePanel 等组件展示执行轨迹。 |
| `lib/outline.ts` | 大纲数据处理。处理分场大纲相关的数据转换和展示逻辑，配合 DetailOutlinePanel。 |
| `lib/usePanelPolling.ts` | 面板轮询 Hook（hook，React 的状态逻辑复用机制）。定时轮询后端，更新各面板的数据，让面板能实时反映生成进度（不轮询就是静态的）。 |
| `lib/utils.ts` | 通用工具函数。放各种零碎的工具函数（格式化、计算等），被各处复用。 |

## 📌 核心认知

frontend 是三层结构：

```
app/         页面（用户访问的网址）
  ↓ 用
components/  组件（页面由组件拼成：workspace 业务面板 + ui 基础控件）
  ↓ 用
lib/         工具（api.ts 通信 + stage.ts 转进度 + types.ts 数据格式）
```

**两个咽喉文件**：
- `lib/api.ts` —— 前后端通信的唯一入口（所有请求都走它）
- `lib/stage.ts` —— 把后端 trace 翻译成用户能看懂的进度

**一一对应关系**：`components/workspace/` 的业务面板和 executor 写作流水线的四个岗位对应——后端有访谈员/故事建筑师/分场员/正文写手，前端就有 ChatPanel/StorylinePanel/DetailOutlinePanel/ScriptPanel 展示它们的产出。
