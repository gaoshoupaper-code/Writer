# 人物信息结构规范

本文档定义 Writer 系统中角色档案的核心信息结构。每个角色一个 Markdown 文件，存放在 `character/{角色姓名}.md`。

## 结构总览

```text
角色档案
├── Identity        身份定位
├── Appearance      外貌特征
├── Personality     性格与内心
├── Relationships   关系网络
└── CurrentState    目前状态
```

共 5 个顶层维度，每个维度只保留关键信息项。

---

## 各维度字段说明

### 1. Identity — 身份定位

回答"这个角色是谁"。

| 字段     | 说明                                       |
| -------- | ------------------------------------------ |
| 姓名     | 角色全名                                   |
| 年龄     | 实际年龄或大致年龄区间                      |
| 身份/职业 | 角色在社会/故事中的定位                     |
| 一句话定位 | 用一句话概括这个人在故事里的功能和特质      |

### 2. Appearance — 外貌特征

回答"看到这个人第一印象是什么"。外表描写应服务于性格，不做无关堆砌。

| 字段       | 说明                              |
| ---------- | --------------------------------- |
| 体型/体态  | 高矮胖瘦、姿态特征               |
| 穿着风格   | 偏好的着装风格，反映什么性格倾向   |
| 标志性特征 | 疤痕、习惯动作、独特表情等辨识点   |

### 3. Personality — 性格与内心

回答"这个人外在是什么样，内心深处想要什么"。

| 字段     | 说明                                |
| -------- | ----------------------------------- |
| 性格     | 角色的性格特质与行为倾向             |
| 核心欲望 | 驱动角色做出选择的最深层动力         |

核心欲望定义了角色最根本的追求方向。

### 4. Relationships — 关系网络

回答"这个人和其他角色之间如何互动"。

| 字段     | 说明                                    |
| -------- | --------------------------------------- |
| 重要他人 | 对角色影响最大的 2-4 个人                |
| 互动模式 | 与每人的关系动态（不是静态标签）         |

每个关系用一两句话描述：这人是谁、对角色的意义、两人互动的特点。

### 5. CurrentState — 目前状态

回答"此时此刻这个角色在哪里、在做什么、经历了什么"。随剧情推进动态更新。

| 字段     | 说明                                       |
| -------- | ------------------------------------------ |
| 所在地点 | 角色当前身处何处                           |
| 人物状态 | 身体和心理的综合状态（受伤、疲惫、焦虑等）  |
| 近期事件 | 最近发生的与剧情相关、对角色有影响的事      |

这部分是动态的，每推进一个章节后由系统自动更新，确保后续写作时角色状态与剧情同步。

```markdown
## 外貌特征

体型/体态：
穿着风格：
标志性特征：

## 性格与内心

性格：
核心欲望：

## 目前状态

所在地点：

近期事件：

## 关系网络

- **{关系人A}**：{关系定位} — {互动模式}
- **{关系人B}**：{关系定位} — {互动模式}
```

## 设计原则

1. **只记关键项**：每个维度 2-5 个字段，足以支撑角色在故事中做出合理行为，不至于空白。
2. **核心欲望是引擎**：核心欲望驱动角色做出选择，是情节推动的根本动力。
3. **关系写动态不写标签**：不用"朋友/敌人"这种静态标签，而是描述两人之间怎么互动、怎么互相影响。
4. **外貌为角色服务**：只写能反映性格的内容，不做无意义堆砌。
5. **状态随剧情更新**：CurrentState 是唯一动态维度，每推进一个章节后自动刷新，保证角色状态与剧情同步。
6. **与系统实现一致**：这 5 个维度与 `CharacterGenerateResponse` 的字段一一对应。
