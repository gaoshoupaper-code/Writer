# executor 文件大地图

> **怎么看这张图**：executor 是后端核心，所有 AI 计算都在这里。
> 它按"四层结构"组织（入口层→领域层→平台层→横切层），依赖**只能从外向内**。
> 下方按层分卡片，每张卡片先告诉你"这层/这组管什么"，再逐个列出文件和作用。
>
> **Phase 7 包化后的关键认知**：writing 域的 agent 装配逻辑（prompt/middleware/subagent/skills/reviewer）
> 已物理迁移进 harness 包（`evolution/harnesses/current/`，见 [evolution 文件大地图](../evolution/文件大地图.md) 的 harness 包章节）。
> executor 内的 writing 域只剩"服务壳 + 领域服务 + 风格子系统"，不再含装配逻辑。
> domain 组织规约见 [domain_convention.md](../domain_convention.md)。

## 🚪 入口层（main + routers/）

> **这层管什么**：接 HTTP 请求、分发给对应的 service、返回结果。
> 这层**只做收发**，不含业务逻辑——收到请求后转交给领域层的 service 处理。

| 文件 | 行数 | 作用 |
|------|------|------|
| `main.py` | 176 | FastAPI 服务入口。创建 app 实例，在 lifespan 里初始化数据库、checkpoint 池、风格模块、各 service，挂载所有路由。是整个 executor 的启动点，所有东西从这里装配起来。 |
| `routers/screenplay.py` | 72 | 剧本生成路由。处理"生成剧本"的 HTTP 请求，转交给 writing 域的 MetaAgentService。是从 main.py 抽出的独立路由（PR-14 重构）。 |
| `routers/character.py` | 24 | 角色生成路由。处理"单独生成角色"的请求（不必走完整流水线），转交给 CharacterService。 |
| `routers/workspaces.py` | 407 | **工作区路由（最大的 router）**。管理工作区的增删查、激活风格、列表等。工作区是用户隔离的基本单位，每个用户的创作在一个独立工作区里。 |
| `routers/threads.py` | 133 | 对话线程路由。管理对话线程——一个工作区下可有多个线程，每个线程是一次完整的创作会话。 |
| `routers/context.py` | 113 | 服务实例容器 + 诊断工具。集中存放各 service 的实例供所有 router 共享（main.py lifespan 调 init_router_context 注入），也含 SSE 活跃连接计数等诊断工具。 |
| `routers/internal.py` | 223 | 内部诊断路由。提供端点供运维和 evolution 调用：trace 查询、A/B 回放、快照刷新通知等。不暴露给普通用户。 |

## 🏛️ 领域层（domains/）

> **这层管什么**：真正的业务逻辑，按"业务域"划分。每个域独立、互不依赖（分层铁律第 2 条）。
> 这层是"做什么"——writing 域定义怎么写剧本，image 域定义怎么生成图。
>
> **两种 domain 模式**（详见 [domain_convention.md](../domain_convention.md)）：
> - **进化型 domain**（writing）：agent 装配在 harness 包，executor 内是薄壳（构建 ctx + 调 pkg.assemble）。
> - **独立型 domain**（image）：agent 装配在 executor 内，不参与进化。

### `domains/writing/` —— 写作流水线服务层（进化型 domain）

> **这组管什么**：写作流水线的"服务外壳"——构建运行时上下文、调用 harness 包装配 agent、编排 SSE 流。
> **Phase 7 包化后**：流水线的 agent 定义（prompt/subagent/middleware/skills/reviewer）已迁进 harness 包，
> 本域不再含装配逻辑，只负责"消费包 + 领域服务 + 风格子系统"。

#### 主服务（域根）

> **这组管什么**：流水线的编排外壳——接收请求、构建 ctx、调包装配、跑生成、编 SSE。

| 文件 | 行数 | 作用 |
|------|------|------|
| `writing/agent.py` | 541 | **MetaAgentService（写作主服务，全域最大）**。接收用户请求，构建 RuntimeContext（含 model/backend/checkpointer/styles），调 harness 包的 `assemble(ctx)` 装配 agent，再调 `run_agent_stream` 跑生成。含 generate（同步）+ generate_stream（SSE 流式）两条路径。还含风格解析（从 styling store 解析 → 填充 ctx.styles）、mock 响应、产物读取。想理解写作请求怎么流转，读它就够。（Phase 7 前叫 meta/agent.py，867 行；M4 瘦身后 541 行，SSE 事件分发下沉到 events.py） |
| `writing/events.py` | 327 | **WritingEventSink（写作 SSE 事件处理器）**。实现 platform.streaming.EventSink 协议，把 LangGraph 事件转成前端 SSE 帧（model_output/tool_call/tool_output 等）。含 writing 专属逻辑：章节号推断、总章数统计、字数计算、storybuilding 完成后派生流程图。是 image 的 _ImageSink 的对称实现。（M4 从 agent.py 抽离） |
| `writing/models.py` | 90 | 写作模型构建器（build_writer_model）。key/base_url/model 来源优先级：显式参数 > evolution 激活配置（经 llm_config_loader 拉取）> 环境变量 > 占位。生产写作和 A/B 测试共用此函数，桌面端配置的 LLM key 经此打通到 executor。 |
| `writing/deepseek_thinking.py` | 226 | DeepSeek 思考模式适配器。封装 DeepSeek "思考模式"（先内部推理再输出）的调用差异，让上层用 DeepSeek 时和用普通模型一样简单。 |

#### 领域服务（expert_agent/services/）

> **这组管什么**：写作域的独立业务能力——角色生成、剧情图谱派生。不走 harness 包，是 executor 内的领域服务。

| 文件 | 行数 | 作用 |
|------|------|------|
| `writing/expert_agent/services/character.py` | 232 | 角色生成服务（CharacterService）。封装角色生成生命周期（构建模型→调 AI→验证文件）。既给故事建筑师当角色库用，也独立对外提供 API。继承 BaseAgentService。 |
| `writing/expert_agent/services/storyline_graph.py` | 504 | **剧情图谱生成器（writing 域第二大文件）**。读取 storyline 索引和各线详情，**纯代码确定性生成**（不靠 AI）一张 mermaid 泳道流程图。把多条交织的剧情线按全局时间序号排序、生成可视化图。解析失败跳过不报错——图是派生视图，不拖累主流程。 |

#### 风格子系统（styling/）

> **这组管什么**：管理用户能选的"创作风格"（文艺风/商业片风等），把风格指引塞给流水线各岗位。
> **M3 风格注入闭环后**：风格从这里解析 → 填充 RuntimeContext.styles → harness 包内 assemble 消费 → 注入各 subagent prompt。

| 文件 | 行数 | 作用 |
|------|------|------|
| `writing/styling/router.py` | 106 | 风格的 API 入口（FastAPI 路由）。定义增删改查风格的接口（列出/创建/更新/给工作区设激活风格）。 |
| `writing/styling/store.py` | 73 | 风格仓库，封装风格的数据库存取。把风格存进 SQLite（带 owner_id 多用户隔离）。 |
| `writing/styling/optimizer.py` | 68 | 风格优化器。把用户给的粗略风格描述打磨成精确的风格指令。 |
| `writing/styling/schemas.py` | 40 | 风格数据格式定义（Pydantic 模型）。定义风格四列字段：meta/storybuilding/detail_outline/writing。 |

> **已迁出 executor 的 writing 组件**（Phase 7 迁进 harness 包 `evolution/harnesses/current/`）：
> - 四岗位子代理（interview/storybuilding/detail_outline/writing）→ `subagents/`
> - 审稿审查器 → `subagents/reviewers/`
> - 全部 middleware（goal/meta_readonly/revision_limit/storyline_single_line_limit 等）→ `middleware/`
> - 全部 prompt（8 个 .md）→ `prompts/`
> - 全部 skill（6 个 SKILL.md）→ `skills/`
> - goal 工具 → `tools/`
>
> 这些是"agent 定义"，executor 通过 `load_current_package()` import 包后消费。详见 [evolution 文件大地图](../evolution/文件大地图.md) 的 harness 包章节。

## 🏗️ 平台层（platform/）

> **这层管什么**：所有领域共用的基础设施。它是 executor 的"地基"——agent 怎么跑、数据怎么存、trace 怎么记、安全怎么管，全在这层。
> 这层**绝对不能依赖领域层**（分层铁律第 1 条），它是公用的，不能反过来沾业务。

### `platform/agent/` —— Agent 运行时 + 中间件库 + 包加载（平台层最核心）

> **这组管什么**：让 AI agent 能跑起来的所有底层能力——运行时隔离、中间件护栏、服务基类、**包加载**。
> **Phase 7 包化 + M2 harness 合并后**：原 `platform/harness/` 的 package_loader/ab_runner 已并入本组（loader.py + ab_runner.py）。

#### 包加载与 A/B 编排（M2 从 harness/ 合并）

> **这组管什么**：加载 harness 包（生产路径）+ A/B 变体隔离执行。

| 文件 | 行数 | 作用 |
|------|------|------|
| `platform/agent/loader.py` | 96 | **Agent 包加载器（生产路径，D8=X 同进程）**。用 importlib 加载 `evolution/harnesses/current/` 作为 Python package，返回含 `assemble(ctx)` 的包模块。进程级缓存，换版本需重启。（M2 从 platform/harness/package_loader.py 迁入） |
| `platform/agent/ab_runner.py` | 121 | A/B 变体隔离执行器（D7=② 子进程隔离）。从 evolution 取快照 tar → 解压临时目录 → 起 worker 子进程跑生成。骨架状态（子进程编排待集成）。（M2 从 platform/harness/ab_runner.py 迁入） |

#### 运行时隔离层（runtime/）

> **这组管什么**：对 DeepAgents 框架的隔离封装。框架升级时只改这三个文件，其余不受影响。

| 文件 | 行数 | 作用 |
|------|------|------|
| `platform/agent/runtime/types.py` | 52 | 类型定义隔离层。定义从 DeepAgents 引入的类型符号（CompiledSubAgent/SubAgent 等），统一入口。 |
| `platform/agent/runtime/backend.py` | 62 | 后端隔离层。封装框架的文件系统后端（FilesystemBackend）。 |
| `platform/agent/runtime/factory.py` | 20 | 工厂隔离层。封装框架的 create_deep_agent，领域层通过它创建 agent。 |

#### 中间件库（middleware/）

> **这组管什么**：agent 执行时的各种"护栏和能力"——追踪、安全、错误恢复、产物校验等。
> **注意**：harness 包内还有一套领域专属 middleware（goal/meta_readonly/revision_limit 等），那是 writing 流水线的护栏，会进化。本目录是**通用平台中间件**，所有 domain 共用。

| 文件 | 行数 | 作用 |
|------|------|------|
| `platform/agent/middleware/trace_middleware.py` | 595 | **执行链路追踪中间件（中间件库最大）**。记录 agent 执行每一步（调了谁、产出什么、耗时），形成完整 trace。是 evolution 的数据来源。**唯一出包的 middleware**（强依赖 TraceRecorder）。 |
| `platform/agent/middleware/path_guard_middleware.py` | 291 | 文件系统路径安全守卫。限制 agent 能写哪些路径，基于写白名单校验，是安全的核心防线。 |
| `platform/agent/middleware/context_assembler_middleware.py` | 239 | 通用上下文组装中间件。给 agent 喂前置信息（把指定文件内容拼进上下文）。 |
| `platform/agent/middleware/artifact_validation_middleware.py` | 175 | 产物文件校验中间件。agent 输出前检查产物文件是否存在、是否符合要求。 |
| `platform/agent/middleware/artifact_prerequisite_middleware.py` | 117 | 前置产物校验中间件。开工前确认依赖的前置产物齐了，不齐拒绝启动。 |
| `platform/agent/middleware/error_recovery_middleware.py` | 149 | 工具调用错误恢复中间件。agent 调工具出错时自动重试，不让单次错误中断整个流程。 |
| `platform/agent/middleware/file_write_serialize.py` | 88 | 文件写操作串行化中间件。把并发的文件写操作排队串行执行，防止并发写损坏。 |
| `platform/agent/middleware/trace_callback.py` | 88 | LangChain 回调处理器。注册 run 层级的父子 agent 关系，配合 trace_middleware 构建调用树。 |
