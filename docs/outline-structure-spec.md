# 大纲结构规范

本文档定义 Writer 系统中 `outline.md` 的核心信息结构。每个线程工作目录下一个 `outline.md` 文件。

## 结构总览

```text
outline.md
├── StoryCore        故事核心
├── Structure        结构骨架
├── MainThread       主线
├── SubThread        次线
└── Interweave       交织点
```

共 5 个顶层维度，全部是创作内容。

---

## 各维度字段说明

### 1. StoryCore — 故事核心

回答"这个故事本质上讲什么"。所有情节都应能追溯到这一层。

| 字段       | 说明                                                         |
| ---------- | ------------------------------------------------------------ |
| 一句话故事 | Logline；一句话讲清主角、核心冲突和故事走向                  |
| 故事前提   | 故事成立的世界设定和前提条件（不同于题材，是故事运转的底层规则） |
| 核心主题   | 故事想探讨的深层命题（不是道德说教，是故事自然呈现的思辨方向） |
| 类型与基调 | 故事类型（悬疑/现实主义/奇幻…）和情感基调（暗黑/温暖/讽刺…） |

### 2. Structure — 结构骨架

回答"故事的整体弧线是什么"。定义大结构，不涉及情节细节。

| 字段         | 说明                                                         |
| ------------ | ------------------------------------------------------------ |
| 结构类型     | 三幕式 / 五幕式 / 英雄之旅 / 群像交叉等；说明选择理由       |
| 关键转折点   | 3-5 个核心转折（触发事件、中点、灵魂暗夜、高潮等），各用一句话描述 |
| 高潮设计     | 高潮场景的核心冲突、参与角色和解决方式                       |

结构类型不是装饰——它决定了节奏和信息密度的分布方式。

### 3. MainThread — 主线

故事的核心叙事线。删掉它，故事不存在。

主线由一系列分段按顺序组成，每一段承载情节推进，段与段之间通过衔接过渡形成因果链。

**每个分段：**

| 字段     | 说明                                         |
| -------- | -------------------------------------------- |
| 分段标题 | 简短概括这一段的阶段特征                     |
| 发生了什么 | 这一段的核心事件，聚焦因果（因为…所以…）   |
| 走向   | 这条线的独立演变轨迹                       |
| 如何衔接 | 这一段如何过渡到下一段——是什么推动了转变，是角色的选择、外部事件还是信息揭露 |

衔接不是简单的"然后"，而是"因此"。每一段的结尾状态必须为下一段提供了启动的理由。

### 4. SubThread — 次线

辅助叙事线，可有多条。删掉它，故事仍成立但会单薄。每条次线有自己独立的角色动机和因果链，不从属于主线。

| 字段   | 说明                                       |
| ------ | ------------------------------------------ |
| 名称   | 次线的标识（如"复仇线""爱情线"）          |
| 驱动角色 | 这条次线由谁推动                           |
| 目标   | 次线角色想要达成什么                       |
| 走向   | 这条线的独立演变轨迹                       |

### 5. Interweave — 交织点

主线与次线发生交汇的关键时刻。交织点是关节——主线和次线在这里互相施压，产生单独一条线无法制造的冲突和张力。

| 字段     | 说明                                               |
| -------- | -------------------------------------------------- |
| 交织描述 | 发生了什么交汇                                     |
| 涉及线索 | 哪些主线/次线在此交汇                               |
| 产生的张力 | 交汇催生了什么新的冲突、转折或信息释放             |

## 设计原则

1. **故事核心是锚点**：所有情节都必须能追溯到 StoryCore。如果一条情节无法对应到核心主题或核心冲突，它就是多余的。
2. **结构服务于故事**：结构类型的选择要有理由，不同结构类型意味着不同的节奏分布。
3. **主线分段递进**：每一段通过"发生了什么"推进情节，通过"如何衔接"向下一阶段过渡。衔接不是"然后"，而是"因此"。
4. **交织产生张力**：交织点是最关键的设计——主线和次线在这里互相施压，产生单独一条线无法制造的戏剧冲突。
5. **与角色档案联动**：故事线中的角色引用 `character/` 目录下的角色姓名；角色的核心欲望驱动主线的目标，关系动态驱动次线和交织点。
6. **灵活适配**：短篇可以只有主线两三个分段；长篇可以多段递进。次线和交织点同理，按需增减。维度是约束不是枷锁。
