# storybuilding-initial

首次故事骨架构建流程。两个硬性约束：
1. **严格只生成第一卷（volume/01.md）**，禁止一次性产出多卷。后续卷由增量 Skill 逐步扩展，每次也只增一卷。
2. **总纲只定起点（A01）和终点（A02）两个锚点**，不得插入中间锚点。中间锚点留给增量扩展逐步添加。

## 工作流程

### 步骤 1 — 读取需求背景

读取 `demand.md`，了解用户需求和创作背景。
- demand.md 由主代理维护，你只能读取，不能写入、编辑或删除。

### 步骤 2 — 读取已有产物

读取工作区中已有的文件（首次构建时可能不存在，属于正常情况）。

### 步骤 3 — 产出首卷骨架

按系统提示词中的内容规范，产出首卷骨架。

写入顺序：
1. `worldview.md`（世界观是底层基础）
2. `character/*.md`（人物在世界观中活动）
3. `outline.md`（总纲：故事核心 + 起终点锚点 + 故事线全局表 + 锚点详情）
4. `volume/01.md`（第一卷纲：初始故事线 + 核心事件 + 钩子追踪）

**最小骨架必须包含：**
- 至少 1 个主角档案
- outline.md 中的起终点锚点（A01 起点 + A02 终点，严格只有两个）
- 故事线全局表（至少 1 条主线）
- 只能第一卷的基本结构（故事线段 + 核心事件 + 钩子）
- 基础世界观框架（至少包含时代背景、核心冲突根源）

其他维度可以留占位符（标注"待后续轮次补充"）。

### 步骤 4 — evolution 评估

完成所有产物写入后，调用 evolution 子代理进行评估。

**调用时在任务描述中附加**：
"本次为首次骨架构建，请侧重评估结构完整性和可扩展性。骨架阶段不要求内容丰富度。"

**通过标准**：总分 ≥ 75 且无"必须修改"级别的核心问题 → 接受当前版本。
低于标准 → 读取 evaluation.md，修订对应产物，再次调用 evolution。

最多调用 evolution 3 次（含首次），超过后系统强制终止。

返回时包含：修订轮数、是否有质量风险。
格式：`修订轮数：1/3\n质量风险：无`

### 步骤 5 — 回复父代理

用纯文本简要汇报：
- 产出了哪些维度的骨架
- 骨架覆盖范围（起终点锚点、首卷故事线、核心人物）
- 评估状态
- 下一步建议（建议使用增量 Skill 继续扩展）

## 注意事项
- 严格只生成第一卷，总纲只定起点（A01）和终点（A02）两个锚点
- 只写入系统提示词中定义的文件路径，不修改 demand.md 或其他文件。
- 维度之间保持一致性：引用其他维度的内容时，确保被引用的内容确实存在。
- ID 编号从 01/001 开始，确保后续增量扩展有空间。
