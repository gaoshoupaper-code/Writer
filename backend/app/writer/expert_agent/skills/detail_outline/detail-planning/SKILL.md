# detail-planning

章节细纲的规划执行流程。每次批量生成 3 章细纲后统一评估。

## 执行流程

当父代理要求从某章起始生成细纲时：

1. 读取卷纲（`volume/volume-XX.md`）中的章节划分和关键转折，确定本批 3 章在卷内的位置和任务。
2. 读取卷纲（`volume/XX.md`）中的故事线信息，明确本批需要推进的线索。
3. 读取人物设定（`character/*.md`），确保人物行为、性格表现和关系互动符合设定。
4. 读取之前已完成的章节细纲（`detail/chapter-XX.md`），确保承接连贯。
5. 基于以上上下文，依次生成 3 章细纲（从指定章节编号 N 开始，生成 chapter-N ~ chapter-N+2）。
6. 将 3 章细纲分别写入 `detail/chapter-XX.md`、`detail/chapter-XX+1.md`、`detail/chapter-XX+2.md`（XX 为两位数字），格式遵循 Prompt 中的 chapter-XX 结构规范。

完成当前批次后**立即返回**，等待父代理的下一个指令。不自行开始下一批的规划。

## evolution 评估（单次）

完成 3 章细纲写入后，调用 `evolution` 评估（**全流程只调用 1 次**）：

1. 调用 `evolution` 子代理评估 3 章细纲的整体质量（节奏衔接、人物一致性、线索推进）。
2. evolution 会读取 `detail/` 下的文件并写入评估报告到 `detail/evaluation.md`，然后返回评分和修改建议。
3. 根据 evolution 返回结果决定下一步：
   - 返回 **"无需修改"** → 直接向父代理返回结果。
   - 返回 **"建议修改"** 或 **"必须修改"** → 读取 `detail/evaluation.md` 中的详细评估报告，根据核心问题和修改建议修订 3 章文件**一次**，**不再二次评估**，直接进入返回步骤。
4. 返回父代理时，回复中包含：**是否执行修订**、**是否有质量风险**、**本批章节编号范围**。
   - 格式示例：`执行修订：否\n质量风险：无\n章节范围：chapter-04 ~ chapter-06`

## 注意事项

- 如果在规划过程中发现卷纲有不合理或需要调整之处，在回复中提出建议，不要自行修改卷纲。
- 3 章之间应保持节奏连贯：第 1 章设悬念/铺垫，第 2 章推进/转折，第 3 章收束/抛新钩。
- 简要说明本次生成的文件路径和主要规划内容。
- 完成当前批次后立即返回，等待父代理的下一个指令。
