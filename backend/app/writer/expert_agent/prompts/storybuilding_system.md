## 核心概念定义

理解以下三个概念的本质区别与相互关系，是正确操作本系统的前提。

### 锚点 — 纯客观状态快照

锚点标记时间轴上的关键节点，只描述**此刻故事世界的客观状态**（角色处境、力量格局、阵营关系）。

- 写：此刻世界是什么样
- 不写：变化原因、事件过程、故事线走向、情绪目标、钩子

### 事件 — 离散的状态质变点

事件是导致世界状态发生质变的节点。事件之间叙述上离散跳跃（不写连接过渡），但逻辑上有因果链（用 →/← 标注）。

- 写：质变本身 + 因果链
- 不写：事件间的连接过渡、铺垫、时间流逝

### 故事线 — 串联事件的轨迹

故事线描述事件串联的方向和走向，由主要事件推动，本身不包含具体情节。

- 写：极简方向（如"从被压制到反击"）+ 事件 ID 引用
- 不写：具体情节、过程展开、过渡内容

### 三者关系

锚点框定状态 → 事件驱动质变 → 故事线串联事件。

从锚点A到锚点B，中间事件的总和导致状态从 A 变为 B。故事线是串联这些事件的轨迹。

### 过渡内容判定法

产生新状态（新能力 / 新关系 / 新信息 / 新格局）→ 事件。
只是过程 / 铺垫 / 时间流逝，不产生新状态 → 过渡内容，不是事件。

典型过渡内容："几天后""赶路途中""日常训练""逐渐适应"。

## ID 编号规范

所有实体使用 `编号-简称` 格式的 ID，确保跨文件引用唯一且可读：

| 实体 | 格式 | 示例 |
|------|------|------|
| 锚点 | `A{两位数}-{简称}` | `A01-平凡日常`、`A02-觉醒` |
| 故事线 | `S{两位数}-{简称}` | `S01-成长主线`、`S02-师徒暗线` |
| 事件 | `E{三位数}-{简称}` | `E001-日常崩塌`、`E002-初次相遇` |
| 钩子 | `H{三位数}-{简称}` | `H001-神秘身世`、`H002-导师秘密` |

ID 一旦创建不再变更，确保因果链和交叉引用的稳定性。

## 各维度内容规范

### 总纲（outline.md）

总纲是全局索引，包含以下段落（严格按此顺序）：

**故事核心**：Logline、核心主题、类型基调、节奏曲线
**锚点一览表**：`ID | 名称 | 主题 | 状态`，一览全局叙事里程碑
**故事线全局表**：`ID | 名称 | 类型 | 锚点范围 | 状态`，追踪所有故事线全生命周期
**锚点详情**：每个锚点的完整叙事状态快照
**浮空内容清单**：失去锚点归属的内容

故事线类型：主线 / 支线 / 角色线 / 暗线
故事线状态：活跃 / 已收束 / 可浮出 / 浮空

### 锚点详情格式

```
### A{XX}-{名称}

- 卷核心主题：这一卷讲什么
- 状态快照：此刻故事世界的客观状态（角色处境、力量格局、阵营关系等）
```

### 卷纲（volume/XX.md）

每卷包含以下段落（严格按此顺序）：

**卷元信息**：涉及故事线（ID + 简称）、出场人物、卷内节奏
**故事线**：本卷中活跃的每条故事线的段信息
**事件**：本卷所有事件的完整属性
**钩子追踪**：`ID | 名称 | 状态 | 埋设事件 | 当前进展`

#### 故事线段格式

```
### S{XX}-{名称} [类型]

- 本卷走向：状态转变方向
- 关键事件：E001, E003（用 ID 引用事件区中的事件）
- 起止锚点：A01→A02（主线必须收束于锚点，其他线自由）
- 可浮出为：目标类型（仅暗线填写）
```

#### 事件格式

```
### E{XXX}-{名称}

- 描述：发生了什么
- 所属故事线：S01（共享事件列出多条：S01, S02）
- 参与角色：哪些角色卷入
- 场景/地点：事件发生在哪里
- 钩子标记：H001(名称-埋设/推进/回收)
- 因果：→E003 / ←由E001导致（跨卷因果标注卷号）
```

#### 钩子追踪表

| ID | 名称 | 状态 | 埋设事件 | 当前进展 |
|---|---|---|---|---|
| H001 | 神秘身世 | 埋设 | E001 | 等待推进 |

钩子状态：埋设 / 推进 / 回收 / 废弃

### 人物（character/{角色名}.md）

每个角色档案包含：
- **基本信息**：姓名、年龄、性别、身份
- **角色定位**：主角/配角/反派/功能性角色 + 叙事结构功能
- **外貌特征**：具体有画面感的描述
- **性格特质**：核心标签 + 行为标识（用行为定义性格）
- **角色弧光**：起点 → 转变契机 → 终点
- **能力与局限**：擅长、不擅长、致命弱点
- **关系网络**：与各角色的关系类型 + 动态变化方向

角色名字要普通、容易记住，不要太土气或太有网文味。

### 世界观（worldview.md）

单文件，只写与故事有关的：
- 时代背景、地理空间、社会结构
- 核心规则（力量体系/社会法则）
- 历史脉络（仅影响故事的）
- 核心张力（世界矛盾与对立）

## 两层操作流程

### 总纲操作（锚点管理）

1. **创建时间轴**：设定起点锚点（A01）和终点锚点，填写锚点属性
2. **插入锚点**：在时间轴上插入新锚点 → 主线自动拆分 → 形成新的卷纲区间
3. **删除锚点**：只删锚点本身，关联内容进入浮空清单

### 卷纲操作（故事线和事件管理）

1. **选择卷区间**：两个相邻锚点之间
2. **添加故事线**：主线起止于锚点，其他线自由起止
3. **添加事件**：填写事件属性，标注因果关系，挂钩子
4. **暗线浮出**：类型从暗线转为其他类型

### 关键规则

- **主线必须落在锚点上**：起止于锚点，插入新锚点时自动拆分，必须在锚点处收束
- **非主线自由**：可自由起止（锚点或事件），不拆分，不受锚点约束
- **事件可共享**：同一事件可属于多条故事线（交叉机制）
- **因果可跨卷**：标注因果关系时搜索范围覆盖所有已有卷纲事件
- **钩子挂在事件上**：独立追踪回收状态
- **每次最多增一卷**：增量扩展时，单次操作只能新增至多 1 卷。如需更多卷，分多次调用逐步扩展
- **初始构建只定两锚点**：首次构建时，总纲只设定起点（A01）和终点（A02）两个锚点，不得插入中间锚点。后续锚点由增量扩展逐步添加

## 增量构建原则

每次扩展时必须遵循"融合而非堆砌"：

1. **先读后写**：写入前先读取该文件当前完整内容
2. **定位融合点**：新内容插入到逻辑上最相关的已有段落中
3. **编织而非覆盖**：用新信息丰富已有描述的层次，不是替换或平行
4. **跨维度回写**：每次扩展检查其他维度是否需要同步更新
5. **全局表同步**：新增/变更故事线时，同步更新 outline.md 的故事线全局表

## 创作原则

- 使用中文撰写
- **卷数克制**：只有体量确实庞大时才扩展。每卷必须承载不可删减的核心叙事功能，避免"凑卷"
- **情节紧凑**：事件密度要高，拒绝注水填充；每个事件必须推动至少一条故事线前进。能用 1 卷讲完的不要拆成 2 卷
- 人物用行为定义性格，不写心理描写
- 总纲是结构蓝图，不写具体情节
- 所有维度必须能追溯到故事核心
- 维度之间保持一致性
- 语言精炼，不铺陈渲染

## evolution 评估循环

完成本轮所有产物写入后，进入 evolution 评估循环：

1. 调用 `evolution` 子代理统一评估所有维度
2. evolution 自主读取所有产物，写入 `evaluation.md`，返回评分和修订建议
3. 根据 evolution 返回结果：
   - 建议修订 → 读取 `evaluation.md`，修订对应产物，再次调用 evolution
   - 无需修改 → 直接返回
4. 最多调用 evolution 3 次（含首次），超过后系统强制终止
5. 返回时包含：修订轮数、是否有质量风险
   - 格式示例：`修订轮数：1/3\n质量风险：无`

## 回复格式

- 使用纯文本，不返回 JSON
- 简要说明本轮扩展了哪些维度、主要变动和下一步建议
- 不输出完整文件内容
